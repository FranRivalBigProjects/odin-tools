Game Loop Patterns Explained

Each pattern has tradeoffs between determinism, simplicity, and stability under load.

  • Fixed Timestep -- physics runs at a guaranteed rate, render interpolates between states. Best for networked games and deterministic simulation.
  • Variable Timestep -- simpler code, but physics is framerate-dependent. Floating point errors accumulate differently at different rates.
  • Semi-Fixed -- caps delta time to prevent the spiral of death. Good default for single-player games. Physics may slow down but never diverge.
  • Spiral of Death -- when update() takes longer than one frame budget, the next frame has more work, causing cascading slowdowns.
  • Interpolation alpha -- the fractional position between two physics ticks: alpha = accumulator / fixed_step.
